\begin{abstract}
This project presents the design and development of an Autonomous Tactical Rover prototype built using a Raspberry Pi and an Arduino Nano. While advanced tactical robots are expensive and hard to access locally, our goal was to build a low-cost version using off-the-shelf components. The Raspberry Pi handles human detection using a YOLOv8 model, runs a Flask web server for the control dashboard, and drives the motors. The Arduino Nano runs the turret scanning and the automated firing sequence. The two units are connected via serial (UART). The rover supports two modes: an autonomous shooting mode that triggers when a human is detected, and a manual override mode for remote steering. Testing shows that real-time human detection and safe automated firing can be achieved on a very low budget.
\end{abstract}

\section{Introduction}
Autonomous ground vehicles are becoming standard in defense and security, allowing operations to be managed from a safe distance. While global research has advanced rapidly, local development of such platforms in Bangladesh is still very limited. Most tactical robots are imported and extremely expensive. 

We wanted to see if a small student team could build a functional, budget-friendly prototype using cheap, widely available hardware like the Raspberry Pi and Arduino Nano. Specifically, our project explores whether a dual-controller architecture can support real-time human detection and automated firing while keeping the firing subsystem safely isolated from the main operating system.

This report is structured as follows. Section II explains our design objectives. Section III and IV cover the system architecture and hardware design. Section V details the software implementation on both boards. Section VI describes the control dashboard. Finally, we discuss our test results, system limitations, and future improvements in Sections VII, VIII, and IX.

\section{Project Motivation}
The project focuses on five main goals:

\subsection{User Safety}
The main purpose is to keep operators out of dangerous areas. Controlling the rover remotely over Wi-Fi allows us to do scanning and shooting tasks from a safe distance.

\subsection{AI-Assisted Detection}
We want to reduce human reaction time using computer vision. The Raspberry Pi runs a YOLOv8 model on the camera stream to quickly identify human presence and trigger the engagement sequence in auto mode.

\subsection{Cheap and Decoupled Hardware}
Instead of using a single expensive computer, we split the tasks between a Raspberry Pi and an Arduino Nano. The Pi does the heavy vision and networking work, while the Arduino Nano handles the fast, timing-sensitive servo sweeps and firing. This makes the hardware cheap and easy to troubleshoot.

\subsection{Local Prototyping}
Since tactical robotics is not commonly developed locally in Bangladesh, this project serves as a proof of concept. It shows that useful autonomous security platforms can be built with affordable parts available in the local market.

\subsection{Manual Overrides}
Even in autonomous mode, safety is crucial. The operator can switch to manual control at any time via the dashboard. We also added an emergency stop button that halts the motors instantly.

\section{System Architecture}
The rover uses a dual-controller setup where the Raspberry Pi handles computer vision and driving, and the Arduino Nano handles the turret. Fig.~\ref{fig:block} shows how the components connect.

The Pi and Arduino are decoupled for safety. The Pi does not control the turret servos or the launcher flywheel motor directly. It only sends a single character ('F') over a serial UART connection when its logic decides to fire. The Arduino Nano continuously polls this serial interface. This makes the firing code simple and isolated from any network lag or OS crashes on the Raspberry Pi.

In AUTO mode, the system follows this simple logic (shown in Fig.~\ref{fig:flowchart}):
\begin{enumerate}
    \item \textbf{Video Capture:} The Pi camera captures frames using the \texttt{picamera2} library.
    \item \textbf{YOLO Detection:} A YOLOv8n model checks each frame. If a person (class 0) is detected with a confidence score above 0.6, the system flags it.
    \item \textbf{Auto Firing:} If the mode is set to AUTO, the Pi halts the wheels immediately and sends the fire command ('F') to the Arduino Nano to trigger the firing mechanism.
\end{enumerate}

\section{Hardware Design}
\subsection{Raspberry Pi}
The Raspberry Pi acts as the central hub. It runs the Flask web server, processes the YOLOv8n model, streams live video, and sends control signals to the motor driver using the \texttt{gpiozero} library. We chose the Pi because microcontrollers cannot handle real-time object detection and web hosting simultaneously.

\subsection{2WD Differential-Drive Chassis}
For movement, we used a standard 2WD chassis with two DC motors. We control the direction (forward, backward, left, right) and speed (10\% to 100\%) by driving the left and right motors at different duty cycles based on inputs from the dashboard.

\subsection{Turret and Firing Assembly}
The turret is managed by the Arduino Nano and contains:
\begin{itemize}
    \item Four servos: Servos 1 and 4 are linked to sweep horizontally, and Servo 2 nods vertically to scan the area. Servo 3 is only used as a physical trigger to push the dart/projectile.
    \item Flywheel launcher: Controlled via an ESC (Electronic Speed Controller) that spins up the launcher motor.
    \item Local feedback: A buzzer sounds during target lock, and a 16x2 I2C LCD displays status messages like ``HUMAN DETECTED'' or ``FIRE!!!''.
\end{itemize}

\section{Software Implementation}
\subsection{Raspberry Pi Software}
The Pi runs a Python script with a Flask web server. A background thread called \texttt{ai\_loop} captures frames from the camera and runs YOLOv8n. To keep frame rates usable, we resize the inference resolution to 256~px. If a human is detected with a confidence above 0.6 and the rover is in AUTO mode, it stops and fires. If the system is in MANUAL mode, the loop still runs but the motors only respond to user dashboard inputs. The Flask app handles routes for steering controls (\texttt{/control/<key>}), speed settings, and streaming the MJPEG camera feed. Listing~\ref{lst:ai_loop} shows the main decision structure.

\begin{lstlisting}[caption={Core AUTO/MANUAL decision logic (simplified from \texttt{ai\_loop}).}, label={lst:ai_loop}, language=Python]
if mode == "auto" and not emergency_stop:
    if human_detected:
        stop()
        fire()      # sends 'F' over serial
    else:
        forward()
elif mode == "manual" and not emergency_stop:
    apply_command()  # last manual key

if emergency_stop:
    stop()
\end{lstlisting}

\subsection{Arduino Firmware}
The Arduino Nano code sweeps the scanning servos continuously to look for targets. At the same time, it checks for serial data. When it receives the character 'F' or 'f' from the Pi, it pauses scanning and runs the \texttt{fireSequence()} function (Listing~\ref{lst:fire_seq}). This function turns on the buzzer, displays a message on the LCD, spins up the flywheel launcher motor via the ESC, sweeps the trigger servo to push the dart, and then resets everything to resume scanning.

\begin{lstlisting}[caption={Fire sequence executed on the Arduino Nano.}, label={lst:fire_seq}, language=C++]
void fireSequence() {
  digitalWrite(BUZZER_PIN, HIGH);
  lcd.print("!! HUMAN !! DETECTED");

  esc.writeMicroseconds(ESC_MAX_US);  // spin up launcher
  servo3.write(0);                     // trigger
  delay(800);
  servo3.write(180);                   // reset trigger
  esc.writeMicroseconds(ESC_MIN_US);   // stop launcher

  digitalWrite(BUZZER_PIN, LOW);
  lcd.print("FIRE COMPLETE / Scanning...");
}
\end{lstlisting}

\subsection{Pi-Arduino Serial Link}
We set up a simple serial UART connection between the Pi and the Arduino Nano. The communication is one-way: the Pi only sends a trigger character when it detects a human. There is no feedback loop or confirmation sent back to the Pi, which keeps the code simple but is a limitation we discuss in Section VIII.

\section{Control Interface}
We built a web-based dashboard (Fig.~\ref{fig:dashboard}) using HTML and JavaScript, hosted directly by the Pi's Flask app. Anyone on the same Wi-Fi network can access it via a phone or PC browser. The dashboard includes:
\begin{itemize}
    \item A live MJPEG video feed from the Pi camera.
    \item Buttons to switch between AUTO and MANUAL modes.
    \item A prominent red Emergency STOP button to halt the motors immediately.
    \item Speed control buttons (10\%, 50\%, 100\%) along with keyboard shortcuts (number keys 1-0 for 10\% to 100\% speed).
    \item On-screen direction buttons and arrow key support for driving.
\end{itemize}

\section{Results and Discussion}
Our testing showed that the prototype worked as planned:
\begin{itemize}
    \item \textbf{Dual-Controller Integration:} The Pi and Arduino communicated reliably over the serial connection. The turret scanned continuously without lagging the drive motors.
    \item \textbf{Real-Time Detection:} Resizing YOLOv8n inputs to 256~px allowed the Pi to run detection with low enough latency to stop and fire in real time.
    \item \textbf{Mode Switching:} Switching between AUTO and MANUAL modes via the dashboard was instant and responsive.
    \item \textbf{Firing Isolation:} Because the firing sequence is fully handled by the Arduino after receiving a single character, it remained reliable even when the Pi experienced CPU spikes.
    \item \textbf{Emergency Stop:} The E-STOP command worked instantly in both operating modes.
\end{itemize}

These results show that you do not need expensive military-grade controllers to build a basic tactical robot. A cheap Raspberry Pi and Arduino Nano are enough to handle real-time detection, driving, and automated shooting.

\section{Limitations}
We found several limitations during our tests:
\begin{itemize}
    \item \textbf{Vision Limitations:} Running YOLOv8n at 256~px resolution makes it sensitive to lighting and distance. It struggles to detect humans far away or in low-light environments.
    \item \textbf{False Positives:} Since the Pi fires based on a single-frame detection threshold (0.6 confidence), a brief false detection can trigger an accidental shot.
    \item \textbf{No Security:} The Flask dashboard and the serial link are unencrypted. Anyone on the same network could access the controls.
    \item \textbf{One-Way Serial:} The Pi has no way of knowing if the Arduino has successfully fired, since communication is strictly one-way.
    \item \textbf{Lack of Navigation Sensors:} The rover only uses the camera to look ahead. It has no ultrasonic or LIDAR sensors, so it cannot detect or avoid obstacles while driving in auto mode.
\end{itemize}

\section{Conclusion and Future Work}
This project shows that a low-cost, dual-controller rover can successfully perform real-time human detection and automated firing. Using a Raspberry Pi paired with an Arduino Nano is a viable, budget-friendly approach to prototyping tactical robotic systems.

For future versions of the rover, we plan to improve:
\begin{enumerate}
    \item \textbf{Multi-Frame confirmation:} Requiring the model to detect a human for several consecutive frames before firing to prevent accidental triggers.
    \item \textbf{Obstacle Avoidance:} Adding ultrasonic or infrared sensors so the rover does not run into walls or obstacles in auto mode.
    \item \textbf{Security:} Adding password login to the web dashboard and encrypting the Pi-to-Arduino serial connection.
    \item \textbf{Two-Way Telemetry:} Updating the serial protocol so the Arduino can send its battery status and fire confirmation back to the dashboard.
\end{enumerate}
